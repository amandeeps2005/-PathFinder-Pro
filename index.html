Dijkstra's Algorithm

Overview

Dijkstra's algorithm finds the shortest path between nodes in a graph with non-negative edge weights. It's a greedy algorithm that maintains a set of vertices whose shortest distance from the source is known.

How It Works

  1. Initialize distances to all vertices as infinite except source (0)
  2. Create a priority queue with all vertices
  3. While queue is not empty:
    • Extract vertex with minimum distance
    • Update distances of adjacent vertices

Time Complexity

O((V + E) log V) using a binary heap, where V is vertices and E is edges.

Use Cases

  • GPS navigation systems
  • Network routing protocols
  • Social networking (shortest path between users)
  • Flight connections

A* Search Algorithm

Overview

A* is an extension of Dijkstra's algorithm that uses heuristics to guide the search towards the goal more efficiently. It combines the actual cost from start (g) with a heuristic estimate to goal (h).

Formula

f(n) = g(n) + h(n)

  • g(n): Actual cost from start to node n
  • h(n): Heuristic estimate from n to goal
  • f(n): Total estimated cost

Heuristics

  • Manhattan Distance: |x1-x2| + |y1-y2|
  • Euclidean Distance: √[(x1-x2)² + (y1-y2)²]
  • Chebyshev Distance: max(|x1-x2|, |y1-y2|)

Breadth-First Search (BFS)

Overview

BFS explores vertices level by level, visiting all neighbors before moving to the next level. It guarantees the shortest path in unweighted graphs.

Algorithm Steps

  1. Start with source vertex in queue
  2. Mark source as visited
  3. While queue is not empty:
    • Dequeue a vertex
    • Process all unvisited neighbors
    • Mark neighbors as visited and enqueue them

Time & Space Complexity

Time: O(V + E) | Space: O(V)

Depth-First Search (DFS)

Overview

DFS explores as far as possible along each branch before backtracking. It uses a stack (or recursion) to remember vertices to visit.

Algorithm Steps

  1. Start with source vertex on stack
  2. While stack is not empty:
    • Pop a vertex from stack
    • If not visited, mark as visited
    • Push all unvisited neighbors to stack

Applications

  • Topological sorting
  • Cycle detection
  • Path finding in mazes
  • Connected components
